# ربات آپلودر تلگرام به S3 (پروژه زحل)

این پروژه یک اسکریپت لینوکسی برای انتقال فایل میان تلگرام و فضاهای ذخیره‌سازی ابری سازگار با S3 است. فرآیند انتقال به صورت استریم آسنکرون انجام می‌شود تا فایل‌ها بدون ذخیره شدن روی هارد دیسک سرور و با کمترین میزان مصرف رم انتقال یابند. سرعت آپلود با استفاده از پچ همزمانی چندبخشی (multi-part concurrent upload) در کتابخانه پایروگرام بهبود یافته است.

## ویژگی‌ها
- انتقال فایل بدون اشغال فضای دیسک (ذخیره مستقیم در RAM به صورت موقت)
- بهینه‌سازی سرعت آپلود به تلگرام با استفاده از ۱۶ کانال موازی
- ابزار مدیریت تحت ترمینال (cli.py یا دستور zohal-up) برای پیکربندی اولیه، مدیریت دسترسی کاربران و بروزرسانی پروژه
- امکان گشت‌وگذار در فایل‌های باکت S3 از داخل چت تلگرام (حذف، تغییر نام و دانلود به تلگرام)
- ساخت لینک دانلود موقت (Signed URL) با تاریخ انقضا
- پشتیبانی از پروکسی برای سرورهای داخل ایران جهت ارتباط با تلگرام

## راهنمای نصب روی سرور (دبیان و اوبونتو)
برای نصب پروژه و تعریف سرویس سیستم‌دی، دستور زیر را در ترمینال سرور اجرا کنید:

```bash
git clone https://github.com/arsamadineh/ZOHAL-TG-UPLOADER.git && cd ZOHAL-TG-UPLOADER && chmod +x scripts/install.sh && sudo ./scripts/install.sh
```

## راه‌اندازی و تنظیمات اولیه
پس از اتمام مراحل نصب، دستور زیر را در خط فرمان سرور خود وارد کنید:

```bash
zohal-up
```

سپس از منوی باز شده گزینه ۷ (Run Setup Wizard) را انتخاب کنید تا تنظیمات مربوط به توکن ربات تلگرام، شناسه عددی اکانت خودتان به عنوان مدیر و مشخصات اتصال S3 از شما گرفته شود. ابزار در انتهای ثبت مشخصات، صحت اتصال را بررسی کرده و در صورت تایید، ربات را به عنوان یک سرویس پس‌زمینه لینوکس اجرا خواهد کرد.

## مدیریت سرویس لینوکس
ابزار zohal-up به عنوان یک واسط برای کنترل سرویس عمل می‌کند، اما می‌توانید به صورت مستقیم نیز از دستورات زیر استفاده کنید:

- بررسی لاگ‌های زنده ربات:
  ```bash
  journalctl -u zohal -f -n 100
  ```
- راه‌اندازی مجدد ربات:
  ```bash
  systemctl restart zohal
  ```
- متوقف کردن موقت ربات:
  ```bash
  systemctl stop zohal
  ```
